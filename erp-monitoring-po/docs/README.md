# Documentation Index

Folder ini mengumpulkan dokumen pendukung project agar dokumentasi tidak tersebar di root aplikasi.

Dokumen utama:

- [README.md](../README.md) sebagai baseline perilaku sistem saat ini
- [ACTION_PLAN.md](./ACTION_PLAN.md) untuk prioritas kerja lanjutan
- [GAP_ANALYSIS.md](./GAP_ANALYSIS.md) untuk analisis gap dan roadmap
- [DEVELOPER_GUIDE.md](./DEVELOPER_GUIDE.md) untuk panduan teknis developer
- [SOP_OPERASIONAL.md](./SOP_OPERASIONAL.md) untuk panduan kerja user operasional
- [STATUS_DICTIONARY.md](./STATUS_DICTIONARY.md) untuk definisi status resmi sistem
- [TEST_SCENARIOS.md](./TEST_SCENARIOS.md) untuk checklist QA dan regression
- [TRANSACTION_SCHEMA_AUDIT.md](./TRANSACTION_SCHEMA_AUDIT.md) untuk audit skema transaksi
- [NORMALIZATION_AUDIT.md](./NORMALIZATION_AUDIT.md) untuk audit normalisasi data
- [UI_UX_PRODUCT_STRATEGY.md](./UI_UX_PRODUCT_STRATEGY.md) untuk arah pengembangan UI/UX dan product evolution
- [assumptions.md](./assumptions.md) untuk asumsi fase awal dan konteks desain

Cara pakai dokumen:

- `README.md` adalah baseline perilaku sistem aktif.
- dokumen lain di folder ini dipakai sebagai panduan pengembangan, refactor, QA, dan arah produk.
- jika ada konflik antara baseline aktif dan dokumen strategi/asumsi, tandai gap-nya lalu putuskan apakah kode akan disesuaikan atau dokumen strateginya yang diperbarui.
